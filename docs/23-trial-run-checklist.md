# UtahLister Trial Run Checklist

Use this once before real outreach so one brand-new item goes through the full
system from intake to posting-ready package.

## Goal

Prove the full UtahLister workflow works cleanly on a fresh item before outreach
starts driving real demand.

## Trial run path

1. Submit the item through the live intake form.
2. Confirm the intake reaches Formspree and the operator inbox cleanly.
3. Create the item record in Notion.
4. Review the photo folder and notes for missing information.
5. Select the pricing lane.
6. Generate:
   - Facebook Marketplace title
   - KSL title
   - short description per platform
   - category recommendation
   - image order
   - buyer reply starter lines
   - 48-hour adjustment move
7. Run human QC using the current checklist.
8. Deliver the final posting-ready package.
9. Post from the seller-owned account or run a guided posting session.
10. Save the live URLs in Notion.
11. Capture first-day performance and proof notes.

## What to verify

- the intake asks for the minimum useful information
- the photos are enough to make a clean draft without guessing
- the sale-goal input and pricing lane logic feel believable
- the listing package is clear enough to copy and post without extra rewriting
- the buyer replies sound natural
- the seller-owned posting model still feels simple in practice

## Success condition

The item should move from intake to posting-ready package without workflow confusion,
missing system steps, or ad hoc decisions that only live in chat history.

## If the trial run feels clunky

Pause outreach and fix the rough edge first. The point of the trial is to surface
workflow friction before public demand does it for us.
