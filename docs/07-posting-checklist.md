# Posting Checklist

Use this checklist the moment you are ready to publish.

## Before login
- confirm item folder is correct
- confirm title, price, and short description are final
- confirm cover photo and image order are final
- confirm selected lane, private floor if provided, and negotiation line are saved
- confirm whether this is seller self-posting or a guided posting session
- confirm the item is allowed on the target platform
- confirm no exact home address, personal email, or primary phone number is being posted by accident

## Default posting model
- seller posts from their own Facebook Marketplace and KSL account
- UtahLister prepares the final listing package and supports the seller through posting when needed
- do not collect or store passwords as the standard workflow
- do not post customer-owned items from a UtahLister-owned marketplace profile as the default service model

## Facebook Marketplace
- upload the selected photos in the chosen order
- use the final Facebook title
- use the final Facebook short description
- confirm category and condition
- confirm pickup settings
- publish

## KSL
- upload the selected photos in the chosen order
- confirm image sizes meet KSL requirements if needed
- use the final KSL title
- use the final KSL short description
- confirm category, condition, payment methods, and location
- publish

## After posting
- copy both live listing URLs
- save URLs to the `Jobs` database
- update `Items` status to `posted`
- update `Jobs` stage to `posted`
- save posted date and posted price
- save the selected lane and beta fee if missing
- capture only the minimum proof needed; do not store raw credentials or unrelated screenshots
- if a guided posting session was used, log out of every client session before leaving the workstation
- if a guided posting session was used, clear temporary downloads, cached login prompts, and saved browser state for the client session

## Client follow-up
Send:
- live links
- buyer reply scripts
- posted price reminder
- 48-hour review note

## 48-hour review
- check views and inquiries
- decide: hold, rewrite lead line, or reprice
- document the decision in Notion

## End-of-job rule
Do not close the job until:
- links are saved
- client has the scripts
- the item is logged in Notion
- logout and session cleanup are complete
