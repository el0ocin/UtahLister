# UtahLister Marketplace Posting Operating Model

## Executive recommendation

Do **not** launch UtahLister with a default model that requires customers to hand
over Facebook or KSL passwords.

Do **not** make UtahLister's own marketplace account the default account for
customer-owned inventory listings.

The safest and most professional V1 model is:
- seller-owned account
- UtahLister-managed listing package
- optional guided posting session
- seller handles pickup, money, and final buyer handoff

KSL's current public guidance reinforces this direction:
- verified profiles improve buyer confidence
- built-in messages protect personal contact info and help KSL investigate abuse
- in-person payment and handoff remain the standard safe pattern

## Why this is the recommended model

### Password sharing creates avoidable risk

If customers give UtahLister their passwords:
- UtahLister takes on account-security risk
- account lockouts and fraud suspicion become more likely
- support becomes messier when login or 2FA issues happen
- blame becomes harder to unwind if something goes wrong

This also conflicts with the general direction of platform-account security
guidance, where users are told not to share passwords and to protect account access.

### UtahLister-owned listings create trust and logistics problems

If the listing is posted from a UtahLister marketplace profile while the item is
physically owned by someone else:
- buyers may assume UtahLister physically has the item
- buyer scheduling gets more complicated
- condition disputes become messier
- pickup coordination becomes more fragile
- the listing identity does not match the true seller relationship

That mismatch is not the cleanest trust signal for a local-pickup business.

## Recommended V1 service model

### Default offer

UtahLister does:
- intake review
- pricing lane recommendation
- title and description
- category recommendation
- image order recommendation
- buyer reply scripts
- 48-hour revision recommendation
- final posting-ready package

Seller does:
- posts from their own Facebook Marketplace and KSL account
- controls the live listing
- answers buyer messages using the provided script, or forwards them if that is
  part of the service tier
- meets the buyer
- collects payment

### Optional V1 upgrade: guided posting

UtahLister can offer:
- a 10-15 minute guided posting session by text, phone, or screen share

In that session:
- the seller logs into their own accounts
- UtahLister guides copy/paste and category selection
- the seller completes the final publish action

This keeps trust and control with the seller while still making the service feel
hands-on and premium.

It also lets the seller benefit from their own account trust signals, such as:
- established marketplace history
- KSL profile identity and verification
- local credibility tied to the real person who will actually meet the buyer

## What UtahLister should not do by default

- collect or store customer passwords
- manage customer 2FA codes as normal workflow
- publish customer-owned items from a UtahLister-owned marketplace account
- represent that UtahLister physically possesses the item when it does not
- collect buyer payments on the seller's behalf in the default model

## Fee collection recommendation

For the beta:
- charge before final delivery of the posting-ready package or guided-posting session

This keeps UtahLister's risk low while making the boundary clear:
- UtahLister sells listing preparation and posting support
- the seller still owns the inventory and the transaction with the buyer

## Best future operating model

### Phase 1
- seller-owned accounts only
- UtahLister creates the listing package
- optional guided posting

### Phase 2
- optional managed buyer-reply support with clear boundaries
- still no routine password collection

### Phase 3
- only evaluate deeper integrations or delegated account access if platform support,
  contracts, security controls, and policy review all make it worth it

## Brand/account recommendation

UtahLister should absolutely have its own brand presence for:
- the website
- intake
- trust and proof
- service marketing
- optional UtahLister service listings

But customer inventory should still be posted from the customer's own seller profile
in the default model.

That means the UtahLister profile can build trust as the service brand, while the
actual item listing still reflects the real local seller who owns the item.

## Positioning language this supports

This model lets UtahLister say something clean and true:

`You keep the item and the payout. UtahLister handles the strategy, listing build,
and posting support that gets it market-ready faster.`

That is easier to defend than claiming UtahLister fully brokers the sale when the
customer still owns the item, meets the buyer, and collects the money.
