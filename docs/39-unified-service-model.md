# UtahLister Unified Service Model

This document is the canonical public-safe definition of what UtahLister sells and how the business is structured.

Use this document as the source of truth when the offer, workflow, outreach, and roadmap need to stay aligned.

## Short Version

UtahLister is a Utah-focused listing service for local sellers.

The business has two equal service paths:

- `New Listing Build`
- `Listing Rescue`

Both services solve the same problem:

`A seller has value sitting in an item, but the listing package is too weak, incomplete, or time-consuming for them to create or improve well on their own.`

UtahLister fixes that by supplying a stronger listing package and a clearer selling path while the seller keeps:

- the posting account
- the item
- the buyer conversation
- the meetup
- the payout

## Core Promise

UtahLister helps Utah sellers earn more from local-sale items by improving the listing package, not by taking over the sale.

That means UtahLister provides:

- pricing guidance
- stronger title and description
- platform-specific listing packaging
- cover-photo and image-order guidance
- buyer reply support
- optional guided posting help

UtahLister does not promise:

- guaranteed sale windows
- guaranteed sale prices
- buyer handling by default
- custody of the item
- default password handoff

## The Two Service Paths

### 1. New Listing Build

This path is for sellers who have not posted yet or who want UtahLister to build the listing package from scratch.

Best fit:

- the seller has photos and basic facts
- the seller has not posted yet
- the seller wants a clean first version instead of trial and error

UtahLister delivers:

- listing package from zero
- pricing lane recommendation
- Facebook Marketplace copy
- KSL copy
- cover-photo and gallery order recommendation
- buyer reply prompts

### 2. Listing Rescue

This path is for sellers who already have a live listing that is stale, weak, underpriced, or confusing.

Best fit:

- listing has sat
- seller already dropped price
- title is generic
- dimensions or logistics are missing
- the item appears better than the listing presentation

UtahLister delivers:

- rescue diagnosis
- revised price lane
- rewritten listing copy
- stronger item framing
- cover-photo and gallery order recommendation
- missing-detail prompts
- relist or edit recommendation

## Relationship Between The Two Paths

This is not two different businesses.

It is one operating model with two entry points.

Shared backbone:

- seller-owned posting accounts
- Utah local-sale focus
- truth-first item presentation
- pricing lanes instead of random price cuts
- delivery package instead of software access
- 48-hour follow-up logic

Main difference:

- `New Listing Build` is creation-first
- `Listing Rescue` is diagnosis-first

## Target Audience

Best-fit sellers:

- Utah individuals and households
- movers and downsizers
- busy sellers who delay posting
- sellers with decent items but weak listing execution
- sellers focused on local pickup rather than shipping

Best-fit item families:

- furniture
- home goods
- bulky household items
- outdoor gear
- boxed consumer items

Lower-fit cases:

- very low-value items with no margin to save
- regulated or policy-sensitive items
- sellers who expect guaranteed outcomes
- sellers who refuse to provide basic factual detail

## Sales And Positioning

Public positioning should stay simple:

`UtahLister helps Utah sellers build or rescue local Marketplace and KSL listings so stronger items stop getting undersold, delayed, or ignored.`

Best message hierarchy:

1. less work
2. clearer listing
3. better pricing discipline
4. seller keeps the account and payout
5. Utah local focus

Do not lead with:

- AI
- automation
- optimization language
- generic marketing jargon

## Delivery Model

Every UtahLister job should end in a ready-to-use listing package.

That package may include:

- recommended price lane
- Facebook Marketplace title and description
- KSL title and description
- cover-photo recommendation
- ordered gallery plan
- buyer reply prompts
- posting or relisting guidance
- 48-hour review rule

## Lead Model

The two paths should be acquired differently.

### New Listing Build

Best channels:

- warm referrals
- partner referrals
- website intake
- local trust surfaces

### Listing Rescue

Best channels:

- manual review of stale local listings
- selective direct outreach
- partner referrals tied to sellers with live listings
- future paid channels once proof exists

Important rule:

`Listing Rescue` is the stronger outbound wedge, but it does not replace the core business.

## Pricing Model

Public pricing can differ by service path, but both paths should stay productized and simple.

Working public-safe structure:

- `New Listing Build`
  - one-item beta pricing by value band
- `Listing Rescue`
  - rescue audit
  - standard rescue
  - bulky or premium rescue
  - custom when needed

The principle is:

`Charge for clearer decision-making and better listing execution, not for access to a tool.`

## Trust Model

This part should not drift.

UtahLister is built on:

- seller-owned accounts
- no default password collection
- no false buyer behavior
- honest item representation
- no hidden defects
- no miracle pricing promises

That trust model applies to both service paths equally.

## Operational Rule

UtahLister remains a service business first.

Private tooling should support:

- intake
- review
- photo handling
- listing generation
- rescue diagnosis
- follow-up
- proof capture

But the customer is still buying:

`a stronger listing outcome`

not:

`software access`
